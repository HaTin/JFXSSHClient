# 多语言（i18n）设计

统一 UI 文案管理：资源 ID + JSON 语言文件 + 运行时切换。

所有 Agent 在新增 / 修改界面文案前必须阅读本文件。

---

## 目标

- 所有界面文案统一用「资源 ID」引用，禁止硬编码。
- 每种语言一个 JSON 文件，按资源 ID 定义文案。
- 切换语言无需重启，实时生效（与主题一致）。
- 缺失文案可回退，不崩溃。

---

## 方案概述

- 资源 ID：点分命名的字符串键（如 `menu.file`）。
- 语言文件：`messages_<locale>.json`，扁平的 `键 -> 文案` 映射。
- I18n 工具：加载当前语言文件，提供 `t(key)` 取文案。
- JavaFX 实时切换：通过绑定（StringBinding）让已显示文本随语言变化自动更新。

---

## 资源 ID 规范

- 全小写，点分层级，按区域命名空间。
- 只用字母 / 数字 / 下划线 / 点。
- 不含具体语言文字。

命名空间约定：

```text
app.*        应用级（标题等）
menu.*       菜单
tree.*       连接树
tab.*        标签页
status.*     状态栏
dialog.*     对话框（dialog.connection.* / dialog.settings.*）
button.*     通用按钮
auth.*       认证方式
error.*      错误提示
```

示例：

```text
menu.file
menu.file.new_connection
dialog.connection.host
button.ok
status.ready
```

## 语言文件

路径：

```text
src/main/resources/i18n/
├── messages_en.json       // English（默认回退语言）
├── messages_zh_CN.json    // 简体中文
└── messages_<locale>.json // 其他语言按需新增
```

命名：

`messages_` + 语言代码（`语言` 或 `语言_国家`，如 `en` / `zh_CN`）+ `.json`。

格式：

扁平 JSON，键为资源 ID，值为文案。

`messages_en.json`：

```json
{
  "app.title": "JFX SSH Client",
  "menu.file": "File",
  "menu.file.new_connection": "New Connection",
  "button.ok": "OK",
  "button.cancel": "Cancel",
  "status.ready": "Ready",
  "dialog.connection.host": "Host",
  "error.connect_failed": "Failed to connect to {0}"
}
```

`messages_zh_CN.json`：

```json
{
  "app.title": "JFX SSH 客户端",
  "menu.file": "文件",
  "menu.file.new_connection": "新建连接",
  "button.ok": "确定",
  "button.cancel": "取消",
  "status.ready": "就绪",
  "dialog.connection.host": "主机",
  "error.connect_failed": "连接 {0} 失败"
}
```

规则：

- 所有语言文件必须包含相同的键集合。
- 占位符用 `{0}`、`{1}`（MessageFormat 风格）。
- 以 `messages_en.json` 为键的基准。

## I18n 调用接口

模块归属：`com.xxx.jfxssh.common.i18n`。

```text
I18n.t(key)              -> 当前语言文案
I18n.t(key, args...)     -> 带占位符替换（MessageFormat）
I18n.tp(key)             -> 返回 StringBinding（用于 JavaFX 绑定，实时切换）
I18n.setLocale(locale)   -> 切换语言并刷新绑定
I18n.currentLocale()     -> 当前语言
```

用法：

```text
// 一次性文案（对话框等每次新建的界面）
label.setText(I18n.t("dialog.connection.host"));

// 需要随语言实时变化的文案（常驻界面：菜单 / 状态栏）
menuFile.textProperty().bind(I18n.tp("menu.file"));

// 带参数
String msg = I18n.t("error.connect_failed", host);
```

## 语言切换

入口：

设置窗口 → General → Language（见 UI_DESIGN.md）。

行为：

- 切换后实时生效，无需重启（常驻文案通过绑定刷新）。
- 选择持久化到 settings 表，key = `language`，value = 语言代码。
- 启动时读取 settings；无记录则用系统语言；都不支持则回退 `en`。

---

## 回退策略

取文案顺序：

1. 当前语言文件命中 → 返回
2. 未命中 → 回退默认语言 `messages_en.json`
3. 仍未命中 → 返回资源 ID 本身，并记 WARN 日志

目的：缺翻译时界面可用、可定位，不崩溃。

---

## 依赖

JSON 解析：Jackson（`com.fasterxml.jackson.core:jackson-databind`）。

实现时加入 pom；本设计阶段不引入。

---

## Agent 开发要求

新增界面文案：

- 必须定义资源 ID，并同步写入所有语言文件。
- 必须通过 `I18n.t` / `I18n.tp` 调用。

禁止：

- 在代码中硬编码界面文字。
- 只改一种语言文件而漏其他语言。
- 自造命名风格的资源 ID（须遵循本文件命名规范）。
